# 正向代理与反向代理

## 简单比喻

| 场景         | 生活类比                                                       |
| ------------ | -------------------------------------------------------------- |
| **正向代理** | 你（客户端）找**中介**代购国外商品，商家不知道你的存在         |
| **反向代理** | 你打公司总机（代理），总机转接到具体员工，你不知道员工的分机号 |

---

## 正向代理（Forward Proxy）

**代理的是客户端**，客户端知道目标服务器，但目标服务器不知道真正的客户端。

```
客户端 ──► 正向代理 ──► 目标服务器
        ↑                   ↑
    知道你               只知道代理
```

### 典型用途

1. **翻墙 / 访问受限资源** — 客户端通过代理访问被封锁的网站
2. **隐藏客户端身份** — 代理掩饰真实 IP
3. **缓存加速** — 代理缓存常用资源，减少带宽
4. **访问控制** — 公司内网通过代理限制员工访问某些网站
5. **审计日志** — 记录所有对外访问

### 常见工具

- **Proxy**（HTTP/SOCKS5）：Shadowsocks、V2Ray、Clash
- **Squid**：传统的缓存代理服务器
- **HAProxy**（正向代理模式）
- **Charles / Fiddler**：调试用 HTTP 正向代理（抓包工具）

---

## 反向代理（Reverse Proxy）

**代理的是服务器**，客户端不知道实际处理请求的是哪台后端服务器。

```
客户端 ──► 反向代理 ──► 后端服务器
        ↑                   ↑
    以为就是服务器         只知道代理
```

### 典型用途

1. **负载均衡** — 分发请求到多台后端服务器
2. **隐藏后端服务器** — 客户端不知道后端真实 IP
3. **SSL 终结** — 代理处理 HTTPS 证书，后端用 HTTP
4. **统一入口** — 将多个微服务统一到一个域名下
5. **缓存静态资源** — 减轻后端压力
6. **WebSocket 代理** — 协议升级和转发

### 常见工具

- **Nginx**：最流行的反向代理 + Web 服务器
- **HAProxy**：专业的 TCP/HTTP 负载均衡器
- **Traefik**：云原生反向代理，自动服务发现
- **Apache httpd**：支持反向代理模块
- **Envoy**：高性能数据面代理（服务网格）
- **Caddy**：自动 HTTPS，配置简洁
- **云 LB**：AWS ALB/NLB、Azure App Gateway、阿里云 SLB

---

## 核心区别

| 维度           | 正向代理                    | 反向代理                      |
| -------------- | --------------------------- | ----------------------------- |
| **代理对象**   | 客户端（Client）            | 服务器（Server）              |
| **谁隐藏了**   | 隐藏客户端                  | 隐藏服务器                    |
| **谁配置的**   | 客户端自行配置              | 服务器管理员配置              |
| **客户端感知** | 客户端知道代理存在          | 客户端不知道代理存在          |
| **典型场景**   | 翻墙、抓包、内网访问外网    | 负载均衡、SSL终结、微服务网关 |
| **代表工具**   | Shadowsocks、Squid、Charles | Nginx、HAProxy、Traefik       |

## 一句话总结

> **正向代理**：替客户端说话，服务器不知道客户端。
> **反向代理**：替服务器说话，客户端不知道服务器。
