# Linux 文件权限速查

## 1. 权限字符串解读

执行 `ls -l` 时看到类似 `-rwxr-xr-x` 的字符串，共 **10 位**：

```
[1] [2~4] [5~7] [8~10]
-   rwx   r-x   r-x
│    │     │     └─ 其他人权限 (others)
│    │     └──────── 所属组权限 (group)
│    └────────────── 文件所有者权限 (user)
└────────────────── 文件类型
```

### 文件类型（第 1 位）

| 字符 | 含义     |
| ---- | -------- |
| `-`  | 普通文件 |
| `d`  | 目录     |
| `l`  | 符号链接 |
| `c`  | 字符设备 |
| `b`  | 块设备   |
| `s`  | 套接字   |
| `p`  | 管道     |

### 权限位（第 2-10 位）

每 3 位一组：`rwx`

| 字符 | 含义           | 数值 |
| ---- | -------------- | ---- |
| `r`  | 读 (read)      | 4    |
| `w`  | 写 (write)     | 2    |
| `x`  | 执行 (execute) | 1    |
| `-`  | 无此权限       | 0    |

## 2. 特殊权限位（SUID、SGID、Sticky Bit）

> **全称**：
>
> - **SUID** = **S**et **U**ser **ID**（设置用户 ID）— 不是 "Super"。
> - **SGID** = **S**et **G**roup **ID**（设置组 ID）。
> - **Sticky Bit** = 粘滞位，无缩写。

### 2.1 SUID — `s` 出现在 owner 的 `x` 位

```bash
-rwsr-xr-x   # 小写 s → 有 SUID + 有执行权限
-rwS-r-xr-x   # 大写 S → 有 SUID + 无执行权限
```

- **含义**: 执行该文件时，进程以**文件所有者**身份运行，而非执行者身份。
- **典型例子**: `passwd`、`ping`、`su`
- **数值**: `4`（加在普通权限前，如 `chmod 4755`）

#### SUID 影响谁？

```
-rwsr-xr-x   root  passwd
  ↑
 SUID
```

| 谁执行        | 执行时的身份       | SUID 是否有影响？ |
| ------------- | ------------------ | ----------------- |
| 所有者 (root) | root（本来就是）   | ❌ 不影响         |
| 同组成员      | 自己（组成员身份） | ✅ 身份 → root    |
| 其他用户      | 自己（other）      | ✅ 身份 → root    |

- SUID **不影响所有者本人**（他本来就以自己身份运行）。
- SUID **不影响 group 和 others 位的执行权限**——他们能不能执行文件仍然由各自的 `r-x` / `r-x` 决定。
- SUID 的唯一作用是：**当其他人执行时，临时把进程身份切换为文件所有者**。

> ⚠️ **关键理解**：SUID 是一个**文件级别的标志位**，不是"只影响 owner 这一列"。
> 只要 owner 位显示 `s`（即使 group/others 位没有 `s`），所有能执行该文件的人（有 `x` 权限）都会被替换身份。
>
> ```
> -rwsr-xr-x
>  └─ SUID 在这里显示，但它影响整个文件
>     同组成员有 r-x → 能执行 → 身份被替成 root
>     其他用户有 r-x → 能执行 → 身份被替成 root
> ```
>
> 类比：就像借了 root 的门禁卡——程序执行期间拥有 root 的权限，跑完卡还回去。

### 2.2 SGID — `s` 出现在 group 的 `x` 位

```bash
-rwxr-sr-x   # 小写 s → 有 SGID + 有执行权限
-rwxr-Sr-x   # 大写 S → 有 SGID + 无执行权限
```

- **文件含义**: 执行时继承**所属组**身份。
- **目录含义**（更常用）: 在该目录下创建的文件/子目录，**继承该目录的所属组**。
- **数值**: `2`（如 `chmod 2755`、`chmod g+s`）

#### SGID 影响谁？

| 谁执行     | 执行时的身份           | SGID 是否有影响？ |
| ---------- | ---------------------- | ----------------- |
| 所属组成员 | 自己（本来就是组身份） | ❌ 不影响         |
| 其他用户   | 身份切换为文件所属组   | ✅ 受影响         |

### 2.3 Sticky Bit — `t` 出现在 others 的 `x` 位

```bash
drwxrwxrwt   # 小写 t → 有 Sticky + 有执行权限
drwxrwxrwT   # 大写 T → 有 Sticky + 无执行权限
```

- **含义**: 只有文件**所有者**（或 root）能删除或重命名自己的文件，即使目录权限是 `777`。
- **典型目录**: `/tmp`
- **数值**: `1`（如 `chmod 1777`、`chmod +t`）

### 2.4 大小写本质：`s`/`S`/`t`/`T` 占的是 `x` 的位置

```
原本的 x 位    设置特殊权限后
   x          →  s   (小写 = 特殊权限 + 原本有 x)
   -          →  S   (大写 = 特殊权限 + 原本无 x)
```

| 显示       | 含义                                                |
| ---------- | --------------------------------------------------- |
| `s` (小写) | 有 SUID/SGID **且** 有执行权限 `x`（正常有效）      |
| `S` (大写) | 有 SUID/SGID **但** **无**执行权限 `x`（通常无效）  |
| `t` (小写) | 有 Sticky Bit **且** 有执行权限 `x`（正常有效）     |
| `T` (大写) | 有 Sticky Bit **但** **无**执行权限 `x`（通常无效） |

> 小写 = 可执行且特殊权限有效；大写 = 不可执行，特殊权限形同虚设。
>
> 修复大写 `S`：`chmod u+x file` 补上 `x`，`S` 就会变回 `s`。

## 3. chmod 数值模式（八进制）

### 3.1 原理

三位八进制数 → 三位二进制 → 对应 rwx：

```
r w x    → 二进制    → 八进制
4 2 1    → 111       → 7
4 2 0    → 110       → 6
4 0 1    → 101       → 5
4 0 0    → 100       → 4
0 2 1    → 011       → 3
0 2 0    → 010       → 2
0 0 1    → 001       → 1
0 0 0    → 000       → 0
```

### 3.2 常用组合速查表

| 数值      | user | group | others | 说明                    |
| --------- | ---- | ----- | ------ | ----------------------- |
| **`777`** | rwx  | rwx   | rwx    | 完全开放（不安全）      |
| **`755`** | rwx  | r-x   | r-x    | 程序文件/目录（默认）   |
| **`750`** | rwx  | r-x   | ---    | 项目组内可读执行        |
| **`700`** | rwx  | ---   | ---    | 仅自己可读写执行        |
| **`644`** | rw-  | r--   | r--    | 普通文本文件（默认）    |
| **`640`** | rw-  | r---  | ---    | 配置文件，组内可读      |
| **`600`** | rw-  | ---   | ---    | 私密文件（如 SSH 私钥） |
| **`400`** | r--  | ---   | ---    | 只读配置文件            |
| **`444`** | r--  | r--   | r--    | 所有人只读              |

### 3.3 特殊权限 + 普通权限组合（四位数字）

| 数值       | SUID | SGID | Sticky | 普通权限  | 示例                |
| ---------- | ---- | ---- | ------ | --------- | ------------------- |
| **`4755`** | ✓    | -    | -      | rwxr-xr-x | SUID 程序           |
| **`2755`** | -    | ✓    | -      | rwxr-xr-x | SGID 目录           |
| **`1755`** | -    | -    | ✓      | rwxr-xr-x | Sticky 目录         |
| **`6777`** | ✓    | ✓    | -      | rwxrwxrwx | SUID+SGID（极危险） |

## 4. chmod 符号模式

```bash
chmod u+x file      # 给 owner 加执行权限
chmod g-w file      # 给 group 去掉写权限
chmod o=r file      # others 权限设为只读
chmod a+rx file     # 所有人加读+执行
chmod u+s file      # 设置 SUID
chmod g+s dir       # 设置 SGID
chmod +t dir        # 设置 Sticky Bit
```

### 操作符

| 符号 | 含义               |
| ---- | ------------------ |
| `u`  | user（文件所有者） |
| `g`  | group（所属组）    |
| `o`  | others（其他人）   |
| `a`  | all（所有人）      |
| `+`  | 添加权限           |
| `-`  | 移除权限           |
| `=`  | 赋予精确权限       |

## 5. 常用操作命令

### 查看权限

```bash
ls -l                     # 查看文件权限
ls -ld /tmp               # 仅查看目录本身权限
stat file.txt             # 更详细的权限信息（含八进制）
namei -l /path/to/file    # 查看路径中每级目录的权限
```

### 修改权限

```bash
chmod 755 script.sh       # 数值模式
chmod u+x script.sh       # 符号模式
chmod -R 755 directory/   # 递归修改
chmod -c file             # 仅输出有变化的部分（verbose）
```

### 修改所有者

```bash
chown user:group file     # 同时修改所有者和组
chown user file           # 仅修改所有者
chgrp group file          # 仅修改所属组
chown -R user:group dir/  # 递归修改
```

## 6. 典型场景速查

| 场景                   | 推荐权限       | 命令                      |
| ---------------------- | -------------- | ------------------------- |
| SSH 私钥（`id_rsa`）   | `600`          | `chmod 600 ~/.ssh/id_rsa` |
| SSH 公钥、配置         | `644`          | `chmod 644 ~/.ssh/*.pub`  |
| SSH 目录               | `700`          | `chmod 700 ~/.ssh`        |
| web 目录               | `755`          | `chmod 755 /var/www/html` |
| 配置文件（含敏感信息） | `640` 或 `600` | `chmod 640 config.yml`    |
| shell 脚本             | `755`          | `chmod 755 deploy.sh`     |
| 共享目录协作           | `2775`         | `chmod 2775 shared_dir/`  |
| `/tmp` 风格目录        | `1777`         | `chmod 1777 /tmp`         |
| 私密日志               | `640`          | `chmod 640 app.log`       |

```
常用三位数（依次为 user/group/others）:
┌───────┬──────────┐
│  7    │  rwx     │
│  6    │  rw-     │
│  5    │  r-x     │

│  2    │  -w-     │
│  1    │  --x     │

└───────┴──────────┘

常用组合: 777 755 750 700  644 640 600 400

特殊权限（四位数的第一位）:
```
