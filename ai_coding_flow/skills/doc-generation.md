# 技术文档生成

## Quick Start

1. 明确目标文档类型（README / API / 架构 / 部署指南）
2. 收集素材（代码、配置、目录结构、对话记录）
3. 按文档类型使用对应模板生成初稿
4. 人工审核：补充缺失的上下文，修正不准确之处

## 常见文档类型与模板

### README

```
# {项目名}

## 简介
{项目做什么}

## 快速开始
{安装、配置、运行}
\`\`\`bash
# 克隆
git clone ...
# 安装
pip install ...
# 运行
...
\`\`\`

## 项目结构
{目录树}

## 测试
{如何运行测试}

## 部署
{部署说明}

## 贡献指南
{如何贡献}
```

### API 文档

```
## {Endpoint}
**{HTTP Method}** \`{URL}\`

### 请求
- Headers: {认证方式}
- Body:
\`\`\`json
{
  "field": "type | 说明"
}
\`\`\`

### 响应
- 200: 成功
\`\`\`json
{响应格式}
\`\`\`

### 错误码
| 状态码 | 含义 |
|---|---|
| 400 | 参数错误 |
| 401 | 未认证 |
```

## Checklist

- [ ] 文档是否有明确的受众定位（开发者/运维/产品）
- [ ] 是否有快速上手的步骤（从 0 到运行）
- [ ] 命令和代码片段是否可复制执行
- [ ] 是否有目录/索引便于快速定位
- [ ] 版本号/日期是否标注

## Rules

- 文档和代码同源：文档中的命令需要实际可执行
- 默认简体中文，代码/命令/标识符保持原文
- API 文档优先以 OpenAPI 规范为基础
