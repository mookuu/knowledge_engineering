# 排障辅助

## Quick Start

1. 收集故障上下文（错误日志、监控指标、用户报告）
2. 提取关键错误信息（traceback、错误码、时间戳）
3. 关联相关代码和最近变更
4. 分析根因（直接原因 + 深层原因）
5. 建议修复方案（临时缓解 + 长期修复）
6. 生成 Postmortem 记录

## 排障流程

```
故障报告
  ↓
收集证据（日志 / 监控 / 复现步骤）
  ↓
AI 分析 → 提出根因假设（多个假设）
  ↓
人工验证假设 → 确认根因
  ↓
AI 建议 → 修复方案 + 临时缓解措施
  ↓
修复实施 + 验证
  ↓
AI 生成 → Postmortem 文档
```

## 排障检查清单

- [ ] 错误日志是否脱敏（不包含用户 PII / 密钥）
- [ ] 是否关联了最近的代码变更
- [ ] 是否排除了基础设施原因（网络、磁盘、证书）
- [ ] 是否排除了配置变更
- [ ] 是否试过复现

## Output Format

```
## 根因
（一句话概括）

## 影响范围
（哪些用户/功能受影响）

## 推荐修复方案
1. 临时缓解（workaround）
2. 长期修复（root cause fix）

## 关联代码
- 文件:行号

## 复现步骤
1. ...

## 预防措施
（如何避免再次发生）
```
