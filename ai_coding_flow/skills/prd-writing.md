# PRD 撰写与细化

## Quick Start

1. 收集原始输入（口头描述、会议纪要、竞品分析、Issue 列表）
2. 用 5W1H 分析需求明确度，列出需要澄清的问题
3. 按标准 PRD 模板生成结构化初稿
4. 检查每个用户故事是否满足 INVEST 原则
5. 补充验收标准（Given/When/Then 格式）

## PRD 模板

```markdown
# PRD: [功能名称]

## 1. 背景与动机
- 为什么做
- 解决什么用户问题
- 相关数据/反馈

## 2. 目标与成功指标
- 业务目标
- 可量化的成功标准

## 3. 功能范围
- In Scope：
- Out of Scope：

## 4. 用户故事
- As a [角色], I want [目标], so that [收益]

## 5. 验收标准
- Given [上下文], When [操作], Then [结果]

## 6. 边界条件
- 空数据、网络异常、权限不足、并发冲突

## 7. 非功能性需求
- 性能、安全、可访问性、国际化
```

## Checklist

- [ ] 每个用户故事是否独立、可协商、可估算、可测试（INVEST）
- [ ] 验收标准是否可客观验证
- [ ] Out of Scope 是否明确
- [ ] 边界条件和异常场景是否覆盖
- [ ] 非功能性需求是否提及

## Output Format

默认输出结构化 Markdown PRD 文档。每个用户故事包含优先级标注（P0/P1/P2）。
