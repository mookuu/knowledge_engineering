# PRD 任务拆解

## Quick Start

1. 读取 PRD 全文，提取所有功能模块
2. 每个模块拆解为用户故事（按优先级排序）
3. 每个用户故事拆解为开发任务（粒度：半天 ~ 2 天）
4. 标注任务间的依赖关系
5. 估算每个任务的复杂度（S/M/L）

## 拆解粒度示例

```
├── 用户故事 1: 用户可以使用邮箱+密码登录 (P0)
│   ├── 任务 1.1: 设计登录页 UI [S]
│   ├── 任务 1.2: 实现登录 API [M]
│   ├── 任务 1.3: 实现 JWT Token 管理 [M]
│   └── 任务 1.4: 实现登录状态持久化 [S]
├── 用户故事 2: 用户可以使用 Google OAuth 登录 (P1)
│   ├── 任务 2.1: 集成 Google OAuth SDK [M]
│   └── 任务 2.2: 处理 OAuth 回调 [S]
```

## Checklist

- [ ] 每个故事是否独立可交付
- [ ] 任务颗粒度是否合理（半人天 ~ 2 人天）
- [ ] 依赖关系是否标注（blocked by 哪个任务）
- [ ] 优先级是否标注（P0/P1/P2）
- [ ] 是否遗漏了非功能任务（配置、部署、文档）

## Output Format

Markdown 层级列表格式，每个任务标注复杂度（S/M/L）和依赖关系。
