# Code Review

## Quick Start

1. 读取代码变更 diff
2. 检查逻辑正确性和边界处理
3. 检查安全性（注入、XSS、密钥泄露）
4. 检查可读性和可维护性
5. 检查测试充分性
6. 按严重级别输出审查意见

## Review Checklist

- [ ] 逻辑正确，覆盖边界场景
- [ ] 无安全漏洞（SQL 注入、XSS、路径穿越）
- [ ] 遵循项目编码规范
- [ ] 函数大小合理，职责单一
- [ ] 错误处理全面
- [ ] 测试覆盖变更代码
- [ ] 无调试代码（print/console.log/TODO/FIXME）
- [ ] 无注释掉的代码

## 严重级别

| 级别 | 标记 | 含义 |
|---|---|---|
| 🔴 **Critical** | 必须修复 | 功能性 bug、安全漏洞、数据问题 |
| 🟡 **Suggestion** | 建议优化 | 可读性、性能、设计改进 |
| 🟢 **Nice to have** | 可选改进 | 风格、注释、命名微调 |

## Output Format

按严重级别分组输出，每条包含：
- 问题描述
- 文件:行号
- 修复建议代码（可选）
