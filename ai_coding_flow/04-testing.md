# 阶段四：测试

## 概述

测试阶段是保证代码质量的核心环节。AI 在此阶段的价值体现在**自动生成测试用例、分析代码覆盖盲区、生成 Mock 数据和测试配置**。AI 生成测试 + 人工审核 = 高效且可靠。

## 核心交付物

| 交付物 | 说明 | AI 可参与程度 |
|---|---|---|
| 单元测试 | 测试单个函数/类的行为 | 高 — 从代码自动生成 |
| 集成测试 | 测试模块间交互 | 中 — 辅助生成 |
| E2E 测试 | 测试完整用户流程 | 中 — 辅助生成脚本 |
| Mock/Stub 数据 | 模拟外部依赖 | 高 — 自动生成 |
| 测试配置文件 | pytest/jest/vitest 配置 | 高 — 从项目自动推断 |

## AI 辅助工作流

```
实现代码完成
    ↓
AI 分析代码 → 识别需要测试的单元 + 边界条件
    ↓
AI 生成测试用例（含正常路径 + 异常路径 + 边界值）
    ↓
AI 生成 Mock 数据（模拟 DB / API / 文件系统）
    ↓
运行测试 → 绿色通过
    ↓
AI 分析覆盖率报告 → 补充缺失场景
    ↓
人工审核 → 确认测试质量
```

### 测试金字塔与 AI 适用度

```
         ┌──────────┐
         │  E2E     │  ← AI 辅助度：中（场景复杂需人工设计）
         │  测试    │
        ┌┴──────────┴┐
        │  集成测试   │  ← AI 辅助度：中（需理解模块间交互）
        │            │
       ┌┴────────────┴┐
       │   单元测试     │  ← AI 辅助度：高（输入输出明确）
       │              │
       └──────────────┘
```

AI 在**单元测试**层面效果最好，因为输入/输出边界明确。E2E 测试需要人工设计业务场景。

## 测试生成策略

### 策略 1：从代码生成测试（白盒）

```
请为以下函数生成 pytest 单元测试，至少覆盖：
- 正常输入路径
- 空值/None 输入
- 边界值（最小值、最大值）
- 异常/错误输入
- 极端情况（超长字符串、负数等）

{代码}

额外要求：
- 使用 pytest 的 fixture 管理依赖
- 使用 pytest.mark.parametrize 覆盖多组参数
- 使用 monkeypatch 模拟外部调用
```

### 策略 2：从接口规范生成测试（黑盒）

```
请根据以下 API 规范生成 pytest 集成测试：

POST /api/users
- 请求: { name: str, email: str, password: str }
- 成功响应: 201 { id: int, name: str, email: str }
- 错误响应: 400 { error: string }

覆盖场景：
1. 创建成功
2. 邮箱已存在（重复）
3. 密码太短
4. 缺少必填字段
```

### 策略 3：从 Bug Report 生成回归测试

```
Bug: 当用户名为空字符串时，登录接口返回 500 而不是 400

请：
1. 为这个场景写一个回归测试
2. 该测试开始时应 FAIL（确认能捕捉到 bug）
3. 修复后应 PASS
```

## 测试质量检查清单

- [ ] 每个测试是否是独立的（不依赖其他测试的执行顺序）
- [ ] 每个测试是否有明确的 Assertion（不只是"没报错"）
- [ ] 是否覆盖：正常路径、异常路径、边界值
- [ ] Mock 对象是否合理（不要 Mock 掉需要测试的逻辑）
- [ ] 测试命名是否清晰（test_当XX时_应返回XX）
- [ ] 是否有 Flaky 测试（有时过有时不过——应修复或删除）
- [ ] 覆盖率是否达到项目标准（通常 70-80%+，非关键代码除外）
- [ ] 测试是否快速（单元测试应毫秒级、集成测试应秒级）

## 注意事项 / 陷阱

- ⚠️ **AI 生成的测试太"快乐路径"** — AI 默认只测正常情况。必须要求它额外覆盖异常和边界
- ⚠️ **测试也在测试 AI** — AI 生成的测试可能包含和代码同样的逻辑错误（同源偏差）。务必人工审核
- ⚠️ **Mock 过度** — AI 容易 Mock 掉关键逻辑，导致测试通过但实际功能是坏的。检查 Mock 的范围
- ⚠️ **覆盖率数字是手段不是目标** — AI 可能生成大量"只为了覆盖"的测试（不含有效断言）。质量 > 数量
- ⚠️ **忽略非功能测试** — AI 默认聚焦功能测试。需要主动要求生成性能/安全/负载测试
- ✅ **最佳实践**：TDD 流程中让 AI 先生成测试再生成实现代码——能显著减少同源偏差
