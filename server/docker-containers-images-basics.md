# Docker 容器与镜像基础概念

> 知识点来源：Reasonix Code 问答整理

## 查看运行中的容器

```bash
# 查看当前运行的容器
docker ps

# 查看所有容器（含已停止的）
docker ps -a
```

## 一、容器（Container）是什么

容器是一个**轻量级的隔离运行环境**：

- 有自己的文件系统、网络、进程空间
- **共享宿主机的操作系统内核**（这是比虚拟机轻量的根本原因）
- 类比：一个独立的"虚拟房间"里跑一个服务

**容器 ≠ 微系统**。更准确地说：容器像一个独立的小型 Linux 环境，但它没有自己的内核。

## 二、Docker 的角色

Docker 是**容器的管理引擎**，负责：

1. **打包（build）** — 把服务 + 依赖 + 基础操作系统打包成**镜像（Image）**
2. **运行（run）** — 从镜像启动**容器（Container）**
3. **编排** — 管理容器的网络、存储、资源限制等

一句话：**容器 = 隔离的运行环境，里面跑一个服务；Docker = 管理这些容器的平台。**

## 三、镜像标签（tag）格式

```
镜像名:标签
nginx:alpine    → nginx 基于 Alpine Linux（~23MB）
nginx:latest    → nginx 基于 Debian（~187MB）
nginx:1.27-alpine → 固定 nginx 版本 + Alpine
```

> `alpine` 不是 nginx 的版本号，是**基础操作系统镜像的标签**。

Alpine Linux 是一个极简 Linux 发行版，基础镜像只有 ~5MB。同一软件往往有多个基础系统的 tag 可选。

## 四、常用基础系统对比

| 基础系统 | 大小 | 包管理 | C库 | 适用场景 |
|---|---|---|---|---|
| **Alpine** | ~5MB | apk | musl | 追求小镜像体积 |
| **Debian Slim** | ~50MB | apt | glibc | 平衡体积与兼容性 |
| **Ubuntu** | ~80MB | apt | glibc | 开发调试、依赖多的服务 |
| **Distroless** | ~0-10MB | 无 | Google 维护 | 生产环境，最小攻击面 |
| **Scratch** | 0MB | 无 | 无 | 编译型语言（Go/Rust）空镜像 |

> **每个容器必须有一个基础操作系统**（哪怕是空的 `scratch`），因为没有操作系统内核的服务没法直接运行。

## 五、镜像（Image）vs 容器（Container）

| | 镜像 | 容器 |
|---|---|---|
| 是什么 | 打包好的只读模板 | 镜像跑起来后的实例 |
| 类比 | 安装包（.exe） | 正在运行的程序 |
| 包含基础 OS？ | ✅ 是 | ✅ 是（镜像里的那份） |
| 包含内核？ | ❌ 否 | ❌ 否 — 用宿主机的 |
| 可写？ | ❌ 只读 | ✅ 可以写 |

### 容器的分层结构

```
┌─────────────┐
│  可写层      │ ← 容器的日志、临时文件、运行时改动写在这里
├─────────────┤
│  nginx 程序  │
├─────────────┤
│  C 库 (musl) │
├─────────────┤
│  Alpine 系统 │
├─────────────┤
│  Linux 内核  │ ← 共享宿主机的，不打包进镜像
└─────────────┘
```

## 六、核心理解

1. **打包时基础操作系统会被打包进镜像** — 这就是 Docker 保证"在任何机器上运行结果一致"的原理
2. **不打包内核**（共享宿主机内核）— 这是容器比虚拟机（VM）轻量的根本原因
3. 基础系统选哪个取决于：镜像大小、安全攻击面、包管理工具、C 库兼容性
4. Alpine 是体积最小的主流方案，**但不是唯一方案** — Debian、Ubuntu、Distroless、Scratch 都是常见选择

## 七、完整工作流

```bash
# 1. 打包 — 把应用 + 基础系统 + 依赖写进镜像
docker build -t my-app .

# 2. 运行 — 从镜像启动一个容器（活的实例）
docker run -d --name my-container -p 8080:80 my-app

# 3. 停止并删除容器
docker stop my-container && docker rm my-container
```
