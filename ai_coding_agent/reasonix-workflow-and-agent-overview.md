# AI Coding Agent — 概念与 Reasonix 工作方式

> 来源：Reasonix Code 问答整理

## 一、什么是 AI Coding Agent

AI Coding Agent（AI 编码智能体）是一类能**调用工具、自主多步推理、持久记忆**的代码辅助工具，不同于传统的一问一答式 AI 助手。

### 核心特征

| 特征 | 说明 |
|---|---|
| **工具调用** | 能读取文件、搜索代码、执行命令、编辑文件等 |
| **自主多步推理** | 不是一问一答，能拆解任务、按步骤执行 |
| **持久记忆** | 跨会话记住用户偏好和项目知识 |
| **可扩展能力** | 通过 MCP 服务器、Skills 技能、插件增强 |

### 常见的同类工具

| 工具 | 定位 |
|---|---|
| **Reasonix Code** | 终端/CLI 编码 Agent |
| **Claude Code** | 终端编码 Agent（Anthropic） |
| **Cursor** | Agent 化 IDE |
| **GitHub Copilot Agent** | IDE 内 Agent 模式 |
| **Devin** | 自助式工程 Agent |
| **OpenAI Codex Agent** | 代码 Agent |

### 术语演变

```
AI Assistant（辅助问答）
    ↓
AI Coding Assistant（代码问答 + 补全）
    ↓
AI Coding Agent（能调工具、自主多步执行、记忆）
    ↑ 当前阶段
```

AI Coding Agent 比传统的"代码助手"多了工具调用、记忆系统和自主执行的能力，但还不到"完全自主的 AI 工程师"级别（中间仍需要人审核/批准）。

## 二、Reasonix Code 工作方式详解

### 2.1 架构分层

```
┌─────────────────────────────────────┐
│         用户输入（问题/指令）       　│
└──────────────────┬──────────────────┘
                   ▼
┌─────────────────────────────────────┐
│   Reasonix（整车 — 车身+系统）  　　　│
│                                     │
│  ├── /new 命令（清空上下文重启会话）　 │
│  ├── 记忆系统（remember/forget）      │
│  ├── Skills 可复用技能                │
│  ├── MCP 服务器集成                   │
│  ├── 文件编辑（SEARCH/REPLACE）       │
│  └── 工具调度（read_file, run_command 等）│
├─────────────────────────────────────┤
│   DeepSeek（发动机 — 推理引擎）         │
│  理解语言、推理、生成回答               │
└─────────────────────────────────────┘
```

| 能力 | 归属 |
|---|---|
| `/new` 命令、会话管理 | **Reasonix** |
| 记忆系统（跨会话持久化） | **Reasonix** |
| Skills 技能系统 | **Reasonix** |
| 文件编辑（SEARCH/REPLACE） | **Reasonix** |
| MCP 服务器集成 | **Reasonix** |
| 工具调用（read_file 等） | **Reasonix** |
| 底层语言理解、推理、回答生成 | **DeepSeek** |

### 2.2 `/new` 命令（重启会话）

`/new` 清空当前上下文，启动一个全新的会话。启动时自动注入以下内容到提示词中：

```
/new ─────────────────────────────────────────────
│ 启动新会话时，系统自动注入：
│ ├── 系统提示词（Agent 身份和规则）
│ ├── pwd（当前项目路径）
│ ├── MEMORY.md（全局记忆索引）
│ ├── HIGH PRIORITY 块（priority:high 的记忆条目）
│ └── Skills 索引
└─────────────────────────────────────────────
```

> **不是关键词触发**，是**全集加载**。所有记忆文件的内容都会注入到提示词中，Agent 一开始就知道这些信息，不需要用户重复解释。

### 2.3 记忆系统

- **`remember`** — 保存一条记忆，跨会话持久化
- **`forget`** — 删除一条记忆
- **`recall_memory`** — 读取完整记忆内容

记忆分两个作用域：
| 作用域 | 存储位置 | 可见性 |
|---|---|---|
| `global` | `~/.reasonix/memory/global/` | 所有项目 |
| `project` | `当前项目/.reasonix/memory/` | 仅当前项目 |

### 2.4 编辑系统：SEARCH/REPLACE

Reasonix 通过 SEARCH/REPLACE 块编辑文件，不是直接改写：

```
path/to/file.ext
<<<<<<< SEARCH
需要查找的唯一原文（必须完全匹配）
=======
替换后的内容
>>>>>>> REPLACE
```

特点：
- SEARCH 文本必须是文件中唯一的（否则拒绝，避免意外重写）
- 编辑前必须 `read_file` 读取过（确保 SEARCH 匹配的是真实内容）
- 支持 `multi_edit` 一次提交多处修改，原子化（一个失败全都不写）

### 2.5 Skills 技能系统

用户可编写可复用的 playbook（SKILL.md），通过 `/skill <name>` 或 `run_skill` 调用。两种运行模式：

| 模式 | 行为 |
|---|---|
| **inline** | 技能内容追加到当前会话上下文 |
| **subagent** | 启动独立子 Agent 执行，只返回最终结果 |

### 2.6 MCP 服务器

通过 MCP（Model Context Protocol）集成外部工具，如：
- `filesystem` — 文件系统操作
- `memory` — 记忆系统
- `github` — GitHub API
- `puppeteer` — 浏览器自动化
- `everything` — 全量工具

## 三、AI Coding Agent 的工作方式特点

### 3.1 与传统 AI 助手的区别

| 对比维度 | 传统 AI 助手（ChatGPT 等） | AI Coding Agent |
|---|---|---|
| 交互方式 | 聊天框 | 终端 + 工具调用 |
| 感知能力 | 只能理解文字 | 可读取文件、执行命令、查看目录 |
| 行动能力 | 无（只能回答） | 能编辑文件、运行命令、创建项目 |
| 记忆持久化 | 仅当前对话 | 跨会话持久记忆 |
| 任务复杂度 | 单轮问答 | 多步推理、拆解执行 |
| 审核机制 | 无 | 部分操作需用户批准 |

### 3.2 典型工作流

```
用户输入请求
    ↓
Agent 拆解任务（多步推理）
    ↓
┌─ 循环 ─────────────────────────┐
│  工具调用（读文件/搜索/执行命令等）  │
│  分析结果                        │
│  决定下一步（继续/编辑/回答）       │
└────────────────────────────────┘
    ↓
输出结果 或 提交编辑
    ↓
（如需）保存记忆供下次使用
```
