# 并发 vs 并行

## 一句话区分

| 概念                    | 核心                       | 比喻                                             |
| ----------------------- | -------------------------- | ------------------------------------------------ |
| **并发（Concurrency）** | **逻辑上**同时处理多个任务 | 一人吃三口饭、回一条消息、再吃三口饭（交替执行） |
| **并行（Parallelism）** | **物理上**同时处理多个任务 | 两个人，一个吃饭一个回消息（真正同时）           |

> 并发是关于**程序结构**的，并行是关于**执行**的。
> — Rob Pike（Go 语言之父）

---

## 图解

```
并发（单核 CPU，时间片轮转）:

  Task A  ██░░░░██░░░░██░░░░
  Task B  ░░██░░░░██░░░░██░░
          ──────────────────→ 时间
          A和B在交替执行，宏观上"同时"进行

并行（多核 CPU，真正同时）:

  Core 1  ████████████░░░░░░
  Core 2  ░░░░████████████░░
          ──────────────────→ 时间
          A和B在各自核心上真正同时运行
```

---

## 详细对比

| 维度                | 并发                           | 并行                         |
| ------------------- | ------------------------------ | ---------------------------- |
| **本质**            | 任务交错执行（interleaving）   | 任务同时执行（simultaneous） |
| **硬件要求**        | 单核即可                       | 必须多核（或多处理器）       |
| **关注点**          | 任务的**结构设计**             | 任务的**执行效率**           |
| **目标**            | 提高响应速度、充分利用等待时间 | 提高吞吐量                   |
| **实现方式**        | 多线程、协程、事件循环         | 多进程、多核并行计算         |
| **Python 中的限制** | GIL 限制多线程 CPU 密集任务    | 多进程不受 GIL 影响          |

## 生活中的例子

### 并发 — 一个人做三件事

```
你只有一个人（单核）：
① 烧水（等待水开）
② 等水开的时候去切菜
③ 切完菜回来水开了，下面条
```

这就是并发：**一个人**通过合理安排任务的**等待时间**，让多件事看起来在"同时推进"。

### 并行 — 三个人各做一件事

```
你有三个人（多核）：
① A 烧水
② B 切菜
③ C 下面条
```

三个人**真正同时**在干活，这就是并行。

## 计算机中的典型场景

| 场景                       | 类型 | 说明                                 |
| -------------------------- | ---- | ------------------------------------ |
| **Web 服务器处理多个请求** | 并发 | 单核也能处理，交替响应各客户端       |
| **浏览器加载页面**         | 并发 | 异步请求图片、CSS、JS，交替获取      |
| **视频渲染**               | 并行 | 多核同时渲染不同帧                   |
| **科学计算（矩阵乘法）**   | 并行 | 将矩阵分块到多核并行计算             |
| **聊天应用**               | 并发 | 同时处理多个用户的消息收发           |
| **数据批量处理**           | 并行 | 将 100 万条数据分到 8 个进程同时处理 |

## Python 中的并发与并行

```
                 Python 任务
                    │
         ┌──────────┴──────────┐
         │                     │
       I/O 密集型           CPU 密集型
         │                     │
         ▼                     ▼
    并发更适合             并行更适合
    threading / asyncio     multiprocessing
         │                     │
         ▼                     ▼
    GIL 对 I/O 影响不大    多进程绕过 GIL
    协程无 GIL 问题        每个进程独立 GIL
```

| 方式                          | 适用                             | 本质 | GIL 影响                 |
| ----------------------------- | -------------------------------- | ---- | ------------------------ |
| **threading**（多线程）       | I/O 密集型（网络请求、文件读写） | 并发 | ❌ GIL 限制 CPU 并行     |
| **asyncio**（协程）           | I/O 密集型，高并发连接           | 并发 | ✅ 无 GIL 问题（单线程） |
| **multiprocessing**（多进程） | CPU 密集型（计算、渲染）         | 并行 | ✅ 绕过 GIL              |

### 代码示例对比

```python
# ── 并发：多线程 I/O（单核即可受益） ──
import threading
import requests

urls = ["https://example.com"] * 10

def fetch(url):
    resp = requests.get(url)     # I/O 等待，释放 GIL
    print(len(resp.text))

threads = [threading.Thread(target=fetch, args=(u,)) for u in urls]
for t in threads: t.start()
for t in threads: t.join()
# 10 个请求交替进行，总耗时 ≈ 单个请求耗时（而不是 10 倍）


# ── 并行：多进程 CPU 计算（需要多核） ──
from multiprocessing import Pool

def heavy_calc(n):
    return sum(i * i for i in range(n))

with Pool(4) as p:               # 4 个进程，分布在多核上真正并行
    results = p.map(heavy_calc, [10_000_000] * 4)
# 总耗时 ≈ 单次耗时 / 4（接近线性加速）


# ── 并发：协程（单线程内交替执行） ──
import asyncio

async def task(name, delay):
    await asyncio.sleep(delay)   # 主动让出控制权
    print(f"{name} done")

async def main():
    await asyncio.gather(
        task("A", 2),
        task("B", 1),
        task("C", 3),
    )

asyncio.run(main())
# 输出顺序: B → A → C（但总耗时 ≈ 3s，不是 6s）
```

## 常见误区

> ❌ "多线程就是并行"
>
> ✅ 多线程在单核上是并发，在多核上**可能**并行（但 Python GIL 限制了 CPU 密集型任务的并行）

> ❌ "并发比并行快"
>
> ✅ I/O 密集型场景并发更好（不浪费 CPU 等 I/O）；CPU 密集型场景并行更快

> ❌ "asyncio 是并行"
>
> ✅ asyncio 是**单线程并发**，靠 await 主动让出控制权来实现任务交替

## 总结

```
               并发                       并行
    ┌─────────────────────┐    ┌─────────────────────┐
    │  任务交替执行        │    │  任务同时执行        │
    │  单核即可            │    │  需要多核            │
    │  关注结构设计        │    │  关注执行效率        │
    │  I/O 密集型首选      │    │  CPU 密集型首选      │
    │  threading/asyncio   │    │  multiprocessing     │
    └─────────────────────┘    └─────────────────────┘
```
