# Codex 记忆体系：短期记忆、长期记忆与持久化上下文注入

## 三层架构

### 层级 1：会话上下文（≈ 短期记忆）

每次对话启动时，上一次对话的内容**不会被加载**。每个新对话都是从头开始的，只保留当前 turn 内累积的上下文。

- 范围：当前会话
- 持久化：❌ 不会跨对话保留
- "已压缩历史"提示是会话内超长时的自动摘要，不是持久记忆

### 层级 2：技能 + 规则（≈ 持久化上下文注入点）✅

每次启动 Codex（无论是新对话还是旧对话），以下内容会被自动注入到系统提示中：

| 注入点 | 路径 | 说明 |
|---|---|---|
| **Rules** | `~/.codex/rules/` | 文本规则，每次启动自动注入。内容通常为权限白名单 |
| **Skills** | `~/.codex/skills/` | `SKILL.md` 在匹配到任务描述时才会按需注入（不是每次都全量注入） |
| **Config** | `~/.codex/config.toml` | 模型、权限等全局设置 |
| **Goals** | `~/.codex/goals_1.sqlite` | 目标状态跨会话持久，但内容是结构化数据（状态机），不是自由文本记忆 |

### 层级 3：Memories（SQLite 持久存储）

`~/.codex/memories_1.sqlite` 是独立的记忆数据库。系统设计上允许代理通过工具接口读写记忆条目，但需要用户主动写入。

## 总结

| 维度 | 说明 |
|---|---|
| **短期记忆** | 当前会话上下文，只在当前 turn 中存在，压缩时存活，新对话消失 |
| **长期记忆** | 跨对话持久化。没有自然语言"记忆"，但有 **Rules / Skills** 作为持久化上下文注入点 |
| **注入时机** | Rules 自动注入；Skills 按需触发（任务匹配到 skill 描述时注入） |
| **持久化数据** | Goals（目标状态）、Memories（结构化记忆条目）、Config（全局配置） |

## 最佳实践

如果想在每次对话启动时都让 Codex 知道某些固定信息（如项目结构、偏好设置），写进 `~/.codex/rules/` 下的规则文件或编写一个 Skill 是最有效的方案。
