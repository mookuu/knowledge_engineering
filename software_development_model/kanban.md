# 看板 (Kanban)

> **看板 = 可视化 + 限制 WIP + 管理流动**。源自丰田生产系统（Taiichi Ohno, 1940s），2000s 被 David Anderson 等人引入软件开发。与 Scrum 不同——Kanban 没有固定迭代周期。

---

## 核心原则

| 原则 | 含义 |
|------|------|
| **从现状开始** | 不要推翻重来，先把你现在的流程可视化 |
| **同意以渐进方式改进** | 小步改变，降低阻力 |
| **尊重当前角色和职责** | 不强制角色变更 |
| **鼓励各层级领导力** | 任何人都可以提出改进 |

---

## 核心实践

```
可视化工作流 ────→ 限制 WIP ────→ 管理流动
     │                  │              │
  To Do            每个列上限      找出瓶颈
  In Progress       3-5 个         消除阻塞
  Done                             持续优化
     │                                 │
     └──── 反馈循环 ──────────────────→│
          (站会、评审、回顾)           │
                                       ↓
                                  持续改进
```

---

## 看板 (可视化面板)

```
┌──────────┐  ┌──────────────┐  ┌──────────┐  ┌──────────┐
│ Backlog  │→│ In Progress  │→│  Review  │→│   Done   │
│   ○ ○ ○  │  │ ○ ○ (WIP: 3)│  │    ○     │  │  ○ ○ ○ ○ │
│   ○ ○    │  │             │  │          │  │          │
└──────────┘  └──────────────┘  └──────────┘  └──────────┘
                ↑ WIP 上限 = 3
                满了就不能再拉新卡片
                这说明下游 Review 阶段是瓶颈
```

## WIP 限制 — 为什么要限制？

| 不限制 WIP | 限制 WIP |
|-----------|---------|
| 每个人同时做 5 件事 | 每个人聚焦 1-2 件事 |
| "快完成"的幻觉 | 真实完成的节奏 |
| 上下文切换成本高 | 深度工作时间多 |
| 瓶颈被隐藏 | 瓶颈被暴露 → 可以解决 |
| 前置时间长 | 前置时间缩短 |

---

## 指标

| 指标 | 含义 | 为什么要看 |
|------|------|-----------|
| **Lead Time** | 从需求产生到交付的总时间 | 预测未来交付周期 |
| **Cycle Time** | 从开始开发到交付的时间 | 衡量工程效率 |
| **Throughput** | 单位时间完成的任务数 | 团队产能 |
| **Cumulative Flow Diagram** | 各状态累积卡数随时间的变化 | 一眼看出瓶颈、WIP 膨胀、交付速率 |

---

## Kanban vs Scrum

| 维度 | Scrum | Kanban |
|------|-------|--------|
| **迭代** | 固定 Sprint (1-4 周) | 无固定迭代，持续流 |
| **角色** | PO / SM / 团队 (必设) | 无强制角色 |
| **变更时机** | Sprint 内锁定 | 随时可拉新任务 |
| **节奏** | Sprint 节奏同步 | 事件驱动 |
| **度量** | Velocity (故事点/Sprint) | Lead Time / Cycle Time |
| **适用场景** | 产品开发（可拆分迭代） | 运维 / 支持 / 持续交付流 |
| **会议** | 固定 5 事件 | 根据团队需要自定 |

---

## 适用场景

- ✅ 运维/支持团队（任务随机到达，不可预测）
- ✅ 持续交付团队（每次提交都可上线）
- ✅ 已有流程但想渐进改进（Kanban 从现状出发）
- ✅ 团队抵制 Scrum 的重流程感
- ✅ Scrum + Kanban 混合（Scrumban）
