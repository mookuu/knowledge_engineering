# 软件开发模式全景图

> 从传统重型流程到现代敏捷与 DevOps，软件开发模式（Software Development Model / SDLC Model）的演进脉络、对比与应用场景。

---

## 发展脉络

```
1970 ─ 瀑布模型
  │
  ├── 1980s ─ V模型（测试与开发的对称）
  │        ─ 螺旋模型（风险驱动迭代）
  │
  ├── 1990s ─ 迭代与增量模型
  │        ─ Scrum（1995）
  │        ─ XP 极限编程（1996）
  │
  ├── 2001  ─ 敏捷宣言发布
  │
  ├── 2000s ─ Kanban（软件开发领域应用）
  │        ─ TDD / BDD 实践推广
  │
  └── 2010s ─ DevOps
           ─ CI/CD 成为基础设施
           ─ Lean Startup / 精益开发
```

---

## 三大阵营对比

### 经典/重型模型

| 模式 | 核心思想 | 适用场景 |
|------|---------|---------|
| [瀑布 (Waterfall)](waterfall.md) | 阶段线性推进，文档驱动 | 需求固定、变更极少（政府/军工） |
| [V模型 (V-Model)](v-model.md) | 开发与测试一一对应 | 高质量要求（航空航天/医疗） |
| [螺旋 (Spiral)](spiral.md) | 每轮迭代评估风险 | 大型高风险项目 |
| [迭代与增量](iterative-incremental.md) | 分轮/分块交付 | 需求尚不完全明确 |

### 敏捷家族

| 模式 | 核心思想 | 适用场景 |
|------|---------|---------|
| [敏捷开发 (Agile)](agile.md) | 短迭代、拥抱变化、持续交付 | 绝大多数现代软件项目 |
| [Scrum](scrum.md) | 固定 Sprint，角色明确 | 团队协作、有明确 PO |
| [看板 (Kanban)](kanban.md) | 可视化流、WIP 限制、无固定迭代 | 运维/持续交付/变更频繁 |
| [极限编程 (XP)](xp.md) | 极致工程实践（TDD/结对/CI） | 需求极不稳定、质量要求高 |
| [TDD](tdd.md) | 先测试后编码，Red→Green→Refactor | 对质量有要求的任何项目 |
| [BDD](bdd.md) | Given-When-Then，业务语言写测试 | 跨角色协作、验收自动化 |

### 现代/DevOps

| 模式 | 核心思想 | 适用场景 |
|------|---------|---------|
| [DevOps](devops.md) | 开发运维一体化，CI/CD | 所有追求快速交付的团队 |
| [精益 (Lean)](lean.md) | 消除浪费，快速交付价值 | 资源有限、需要快速验证 |

## 核心维度对比

| 维度 | 瀑布 | 敏捷/Scrum | DevOps |
|------|:----:|:----------:|:------:|
| **迭代周期** | 项目结束（数月~年） | 1-4 周 Sprint | 分钟~天（持续） |
| **需求变更** | 严格冻结 | 欢迎变化 | 随时引入 |
| **文档** | 重文档 | 可工作软件 > 文档 | 自动化文档 |
| **测试** | 最后阶段 | 每个 Sprint | 持续测试 |
| **交付** | 一次交付 | 每次 Sprint 交付 | 持续部署 |
| **反馈循环** | 年/月级 | 周级 | 分钟/小时级 |
| **风险暴露** | 晚（测试阶段） | 早（每次迭代） | 实时 |
| **团队结构** | 职能竖井 | 跨职能团队 | 开发+运维融合 |
| **工具链** | 各自独立 | CI + 看板 | CI/CD 全自动化 |

## 敏捷 vs 瀑布：何时选什么

| 条件 | 倾向瀑布 | 倾向敏捷 |
|------|:-------:|:-------:|
| 需求明确程度 | 完全明确 | 可能变化 |
| 项目规模 | 大型、固定合同 | 中/小型、可协商 |
| 合规要求 | 高（需要完整文档审计） | 中/低 |
| 客户参与 | 仅在里程碑 | 持续参与 |
| 技术风险 | 低 | 高（需要快速验证） |

## 各模式核心特点速查

| 模式 | 关键词 | 一句话总结 |
|------|--------|-----------|
| [瀑布](waterfall.md) | 顺序、文档、不可逆 | 一步一个脚印，完成一个再下一个 |
| [V模型](v-model.md) | 对称、测试先行设计 | 开发怎么想，测试就怎么测 |
| [螺旋](spiral.md) | 风险驱动、原型 | 每次循环先问：最大的风险是什么？ |
| [迭代增量](iterative-incremental.md) | 小步快跑、逐步完善 | 先有骨架再长肉 |
| [敏捷](agile.md) | 拥抱变化、价值驱动 | 人在流程之上，交付胜于文档 |
| [Scrum](scrum.md) | Sprint、站会、回顾 | 让每一周都有可演示的成果 |
| [Kanban](kanban.md) | 可视化、WIP、流动 | 别催，看板上的卡片自己会走 |
| [XP](xp.md) | 结对、TDD、持续集成 | 把工程实践做到极致 |
| [TDD](tdd.md) | 红→绿→重构 | 代码还没写，测试先失败 |
| [BDD](bdd.md) | Given/When/Then | 用业务语言说清楚系统该怎么做 |
| [DevOps](devops.md) | CI/CD、自动化、监控 | 你写代码，流水线帮你发 |
| [精益](lean.md) | 消除浪费、延迟决策 | 不做不需要的东西 |
