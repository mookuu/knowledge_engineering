# Scrum

> **1995 年** Ken Schwaber 和 Jeff Sutherland 在 OOPSLA 会议上首次提出。敏捷家族中最广泛使用的过程框架。

---

## 核心三件套：角色 × 事件 × 工件

```
┌─────────────────────────────────────────────────────┐
│                   SCRUM 框架                         │
├─────────────────────────────────────────────────────┤
│  角色 (Roles)    事件 (Events)    工件 (Artifacts)   │
│  ───────────    ─────────────    ────────────────    │
│  Product Owner   Sprint           Product Backlog    │
│  Scrum Master    每日站会         Sprint Backlog     │
│  开发团队         Sprint Review    Increment         │
│                  Sprint Retro                        │
└─────────────────────────────────────────────────────┘
```

---

## 角色

| 角色 | 职责 | 不做什么 |
|------|------|---------|
| **Product Owner** | Product Backlog 优先级排序，代表客户/业务价值 | 不指挥开发团队如何实现 |
| **Scrum Master** | 保障 Scrum 流程，移除障碍，教练团队 | 不是项目经理，不分配任务 |
| **开发团队** | 3-9 人跨职能自组织，共同对 Sprint 结果负责 | 不接受外部直接指派任务 |

---

## 事件（固定时间盒）

| 事件 | 时长 | 目的 |
|------|------|------|
| **Sprint** | 1-4 周（固定） | 产生一个可交付增量 |
| **Sprint Planning** | ≤ 8h（月 Sprint） | PO 讲解优先级 + 团队拆任务 → Sprint Backlog |
| **每日站会** | 15 分钟 | 同步三件事：昨天做了啥？今天要做啥？有什么阻碍？ |
| **Sprint Review** | ≤ 4h（月 Sprint） | 向干系人演示增量，获取反馈 |
| **Sprint Retrospective** | ≤ 3h（月 Sprint） | 团队反思：哪些做得好/差/做什么改变 |

---

## 工件

| 工件 | 归属 | 特性 |
|------|------|------|
| **Product Backlog** | PO 维护 | 按优先级排序的需求列表，永不完全 |
| **Sprint Backlog** | 开发团队 | 当前 Sprint 要完成的 Backlog 子集 + 实现计划 |
| **Increment** | 团队共同 | 符合 DoD 的可交付成果，是"完成的"而非"快完成的" |

---

## DoD (Definition of Done) — 完成定义

每个团队必须明确定义"做完"。例如：

```
✅ 代码通过 Code Review
✅ 单元测试覆盖率 ≥ 80%
✅ 集成测试通过
✅ 文档更新
✅ 功能在 Staging 环境可演示
```

没有 DoD 的 Scrum 不是 Scrum——DoD 是透明性的基石。

---

## 常见反模式

| 反模式 | 为什么是问题 |
|--------|------------|
| **SM 同时是 PO** | 利益冲突：SM 要保护团队流程，PO 要推动交付 |
| **站会变成汇报会** | 站会是团队相互同步，不是给领导汇报 |
| **Sprint 中塞入额外任务** | 破坏 Sprint Goal 和团队承诺 |
| **Retro 走过场** | 最大的改善机会被浪费 |
| **DoD 形同虚设** | 产生"假完成"的技术债 |

---

## 适用场景

- ✅ 跨职能团队（开发+测试+设计在一队）
- ✅ 需求会在开发过程中学习的项目
- ✅ 需要定期向干系人展示进展
- ✅ 团队愿意并能够自组织
- ❌ 团队不愿或不能定期参加 Scrum 事件
- ❌ 组织文化不允许自组织（微管理）
